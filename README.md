## 💎 Les Piliers de votre Succès

### 🤖 1. L'IA au Service de vos Clients (BudTender IA Expert)
Transformez chaque visiteur en acheteur grâce à notre conseiller **BudTender** disponible 24h/24.

## BudTender — IA Vocale Expert CBD

### Flux architectural

```
Micro navigateur (16kHz PCM)
        │
        ▼
useGeminiLiveVoice.ts ──► Edge Function: gemini-token
        │                  (token éphémère + system prompt + tool declarations)
        │
        ▼
Gemini Live WebSocket (BidiGenerateContent)
        │   tool calls
        ▼
Handlers dans useGeminiLiveVoice.ts ──► Supabase / callbacks UI
        │
        ▼
Réponse audio BudTender (24kHz PCM) ──► AudioContext
```

### Prompt System (`src/lib/budtenderPrompts.ts`)

Architecture modulaire — `getVoicePrompt()` compose ces fonctions privées :

| Fonction | Contenu |
|---|---|
| `_buildIdentity()` | Nom IA, nom boutique, rôle et posture |
| `_buildAnalysisProtocol()` | Règles de raisonnement silencieux |
| `_buildResponseLogic()` | Structure adaptative, règles TTS |
| `_buildGoldenRules()` | Anti-hallucination, upsell, récupération d'hésitation, bundle |
| `_buildToolsTable()` | Référence des outils disponibles |
| `_buildClientContext()` | Données utilisateur personnalisées |
| `_buildCatalog()` | 25 produits max de l'inventaire actuel |

**`VOICE_FORMAT_RULES`** : interdit le markdown, les emojis, les puces et tout caractère qui serait lu par la synthèse vocale. Tout output doit être du français oral naturel.

### Déduplication des tool calls

Fenêtre de 2500ms — les appels identiques dans cette fenêtre sont ignorés silencieusement pour éviter les doubles exécutions dues à la latence réseau.

## Schéma Supabase

### Tables principales

| Table | Description |
|---|---|
| `products` | Catalogue (`id, name, price, stock, category_id, is_active, embedding`) |
| `categories` | Hiérarchie 3 niveaux (`id, name, parent_id, depth, slug`) |
| `orders` | Commandes |
| `order_items` | Lignes de commande |
| `cart_items` | Panier persistant |
| `favorites` | Liste de souhaits |
| `reviews` | Avis produits |
| `loyalty_transactions` | Historique points |
| `referrals` | Liens et codes de parrainage |
| `promo_codes` | Codes promo (`code, discount_percent, expires_at, max_uses, is_active`) |
| `stock_alerts` | Alertes retour en stock (`user_id, product_id`) |
| `blog_posts` | Contenu RAG pour la recherche experte |
| `ai_response_cache` | Cache réponses IA |
| `analytics_events` | Événements de tracking |

### Fonctions RPC clés

| Fonction | Rôle |
|---|---|
| `match_products(embedding, threshold, count)` | Recherche vectorielle sémantique |
| `process_checkout(...)` | Checkout atomique (décrément stock + création commande) |

### Hiérarchie des catégories

3 niveaux optionnels : Racine (depth=0) → Sous (depth=1) → Sous-sous (depth=2)

Migration : `supabase/20260320_category_hierarchy.sql`

Utilitaires dans `src/lib/categoryTree.ts` :
- `buildCategoryTree(flat)` — tableau plat → arbre
- `getCategorySubtreeIds(id, flat)` — tous les IDs descendants pour les requêtes filtrées
